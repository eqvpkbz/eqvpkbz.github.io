LISLISLIS(最长上升子序列)

求长度

dpdpdp - O(n2)O(n ^ 2)O(n2)

动态规划的做法

令f[i]f[i]f[i]表示以第iii个元素结尾的LISLISLIS长度

则有: f[i]=max(f[i],f[j]+1),(a[j]<a[i],j<i)f[i] = max(f[i],f[j] + 1),(a[j] < a[i],j < i)f[i]=max(f[i],f[j]+1),(a[j]<a[i],j<i)

通过枚举f[i]f[i]f[i]与f[j]f[j]f[j]来不断转移状态，然后不断枚举更新最大值

贪心+二分O(n log2 n)O(n \ log_2 \ n)O(n log2​ n)

贪心的做法

维护一个单调栈，然后根据栈定元素元素和当前元素作比较来选择最优策略，定义stack[]stack[]stack[]为单调栈，栈顶元素为stack[top]stack[top]stack[top]，当前序列的第iii个元素为a[i]a[i]a[i]

  1. 如果stack[top]<a[i]stack[top] < a[i]stack[top]<a[i] 那么满足单调，可以直接压入栈中
  2. 如果stack[top]⩾a[i]stack[top] \geqslant a[i]stack[top]⩾a[i] 那么这个时候插入就不满足单调了，那么我们考虑在单调栈中进行二分查找，然后找到第一个stack[j]⩾a[i]stack[j] \geqslant a[i]stack[j]⩾a[i]进行替换即可

Q:为什么进行替换这一贪心的策略是可行的?

A: 因为这样做并没有增长栈的长度，而且这么一接下去就可以有更好的方案，其实感性的理解就是一个在栈中不止一个序列，可以理解为两条或者更多，但是在替换之后的元素的压入可以应用到每一条序列中作出贡献，比如下面的这个例子

LCSLCSLCS(最长公共子序列)

求长度

O(nm)O(nm)O(nm)

动态规划做法，其实原理很简单，就是看当前的位上是否匹配的问题，然后根据这个来转移状态

设有长度为nnn的串SSS与长度为mmm的串TTT，用f[i,j]f[i,j]f[i,j]表示SSS串前iii个字符与TTT串前jjj个字符的LCSLCSLCS则有:

f[i,j]=max(f[i−1,j],f[i,j−1],(f[i−1,j−1]+1)∗[Si=Tj])f[i,j] = max(f[i - 1,j],f[i,j - 1],(f[i - 1,j - 1] + 1) * [S_i = T_j])f[i,j]=max(f[i−1,j],f[i,j−1],(f[i−1,j−1]+1)∗[Si​=Tj​])

也就是说，无论如何，一定有

f[i,j]=max(f[i−1,j],f[i,j−1])f[i,j] = max(f[i - 1,j],f[i,j - 1])f[i,j]=max(f[i−1,j],f[i,j−1])

讨论特殊情况:当Si=TjS_i = T_jSi​=Tj​时

f[i,j]=max(f[i,j],f[i−1,j−1]+1)f[i,j] = max(f[i,j],f[i - 1,j - 1] + 1)f[i,j]=max(f[i,j],f[i−1,j−1]+1)

所以我们可以从这推出结果，然后f[n,m]f[n,m]f[n,m]就是最后的答案

因为f[i]f[i]f[i]总是会从f[i−1]f[i - 1]f[i−1]这一维度转移过来，所以我们可以考虑用滚动数组来优化空间复杂度
